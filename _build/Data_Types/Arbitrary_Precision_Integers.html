Arbitrary Precision Integers

In addition to the regular integer and floating point data types, Julia supports arbitrary precision arithmetic.

Arbitrary precision integers

The BigInt data type represents arbitrarily large integers. You can create a BigInt by passing an integer:

BigInt(1234)

1234

But note that the number (1234 in this example) will first be interpreted as an Int64, and then converted to a BigInt. This means you cannot create larger numbers than what Int64 supports using this syntax:

BigInt(typemax(Int64) + 1)    # Incorrect - overflow

-9223372036854775808

However, expressions involving BigInt objects will automatically be evaluated using the higher precision:

BigInt(typemax(Int64)) + 1    # Correct - evaluated using higher precision

9223372036854775808

Another option is to create the BigInt object using a string syntax, which supports arbitrary lengths:

parse(BigInt, "12345678901234567890")

12345678901234567890

Since Julia automatically will use the higher precision arithmetics for BigInt objects, many of the existing functions and user functions will work without any changes:

factorial(BigInt(30))            # Would have given overflow with Int64

265252859812191058636308480000000
